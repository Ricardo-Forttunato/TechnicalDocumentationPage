Introdução

JavaScript é uma linguagem de script multiplataforma orientada a objetos. É uma linguagem pequena e leve. Dentro de um ambiente host (por exemplo, um navegador web), o JavaScript pode ser conectado aos objetos de seu ambiente para fornecer controle programático sobre eles.

JavaScript contém uma biblioteca padrão de objetos, como Array, Date e Math, e um conjunto básico de elementos de linguagem, como operadores, estruturas de controle e instruções. O JavaScript principal pode ser estendido para diversos propósitos, complementando-o com objetos adicionais; por exemplo:

  • O JavaScript do lado do cliente estende a linguagem principal, fornecendo objetos para controlar um navegador e seu Document Object Model (DOM). Por exemplo, as extensões do lado do cliente permitem que um aplicativo coloque elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e navegação na página.

  • O JavaScript do lado do servidor estende a linguagem principal, fornecendo objetos relevantes para a execução do JavaScript em um servidor. Por exemplo, as extensões do lado do servidor permitem que um aplicativo se comunique com um banco de dados, forneça continuidade de informações de uma invocação para outra do aplicativo ou execute manipulações de arquivos em um servidor.

javaScript e java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em outros. A linguagem JavaScript se assemelha ao Java, mas não possui a digitação estática e a verificação de tipo forte do Java. JavaScript segue a maior parte da sintaxe de expressão Java, convenções de nomenclatura e construções básicas de fluxo de controle, razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de classes em tempo de compilação do Java construído por declarações, o JavaScript suporta um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores numéricos, booleanos e de string. JavaScript tem um modelo de objeto baseado em protótipo em vez do modelo de objeto baseado em classe mais comum. O modelo baseado em protótipo fornece herança dinâmica; isto é, o que é herdado pode variar para objetos individuais. JavaScript também oferece suporte a funções sem quaisquer requisitos declarativos especiais. Funções podem ser propriedades de objetos, executadas como métodos de tipo flexível.

JavaScript é uma linguagem de formato muito livre em comparação com Java. Você não precisa declarar todas as variáveis, classes e métodos. Você não precisa se preocupar se os métodos são públicos, privados ou protegidos e não precisa implementar interfaces. Variáveis, parâmetros e tipos de retorno de função não são digitados explicitamente.

Variáveis

Você usa as variáveis como nomes simbolicos para os valores em sua aplicação.

Um indentificador do JavaScript deve iniciar sempre com uma letra, underline (_) ou sinal do dolar ($); caracteres subsequentes podem também ser os digitos(0-9). Devido ao fato do JavaScript diferenciar caracteres maisculos e minusculos, letras incluem os caracteres de "A" até "Z" (Maisculos) e de "a" até "z" (Minusculos).

Você pode usar a ISO 8859-1 ou letras Unicode como å ou ü em identicadores, também é possivel usar escape-sequences em identicadores. Alguns exemplos de nomes corretos são Number_hits, temp99, and_name.

Declaração de Variáveis

Você pode declarar uma variável em três formas:

Com a palavra chave var. Por exemplo:

1> var x = 42

Esta sintaxe pode ser usada para declarar tanto variavel global quanto local.

Simplesmente atribuindo a ela um valor. Por Exemplo:

1> x = 42

Issso sempre declara uma variavel global. Isto gera um aviso estrito do JavaScript. Você não deve usar essa variante.

Com a palavra chave let. Por exemplo:

1> let y = 13

Esta sintaxe pode ser usada para declarar uma variavel local com escopo de bloco. Veja o escopo de variavel abaixo

Escopo da Variável

Quando você declara uma variável fora de qualquer função, isso é chamado uma Variável Global, porque isto é disponivel em qualquer outro codigo dentro do atual documento. Quando você declara uma variável dentro de uma função, isto é chamado de variavel local, porque ela esta disponivel somente dentro da função.

O javaScript antes do ECMAScript não tinha espoco de bloco; Em vez disso, uma variavel declarada dentro de um bloco era local para a função (ou tinha escopo global) que reside dentro do bloco, por exemplo, o codigo a seguir ira registrar 5 por que o escopo de x é a função (ou contexto global) na qual x foi declarada, não o bloco, em que neste caso é uma declaração if

1> if (true){ var x = 5} console.log(x); //5

Este comportamento muda quando usamos a declaração let introduzida no ECMAScript 2015.

1> if (true){let y = 5;} console.log(y) //ReferenceErro: y is not defined.

Variáveis Globais

Variaveis Globais são de fato propriedades do objeto global. Em páginas web o objeto global é a janela, então você pode definir e acessar variaveis globais usando a janela.variável sintaxe.

Consequentemente, você pode acessar as variáveis declaradas em uma janela ou quadro espcificando pelao nome da janela ou quadro. Por exemplo, se uma variavel chamada phoneNumber é declarada em um documento, você pode se referir a essa variavel a partir de um quadro como parent.phoneNumber.

Declaração If...Else

Use a Declaração if para executar declarações de verdadeiro e falso. Use a calusula opcional else para executar uam declaração se a condição for false. Uma declaraÇão if se parece como a seguri:

if (condition) {statement_1; } else{statemenet_2; }

Condition pode ser qualquer expressão que retorna um valor booleano. Veja Boolean para mais informações. Se a Condição if for true statement_1 é executado, se if for false, statemenet_2 que será executado. statement_1 e statemenet_2 podem ser qualquer declaração, incluindo declarações if aninhadas.

Você também pode compor declarações usando else se houver multiplas Condições testadas em sequencia, como a seguir:

if (condition_1){statement_1; } else if (condition_2){ statemenet_2; } else if (condition_n){ statement_n; } else{statement_last; }

No caso de multiplas Condições somente a primeira Condição lógica ira avaliar se a condição verdade será executada. Para executar multiplas declarações, agrupe-as dentro de uma declarção em bloco ({...}). Em geral, isto é boa prática sempre usar declarações em bloco, especialmente quando aninhar declarações if:

if (condition){statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

Aconselhamos não usar atribuições em uma expressão condicional, por que a atribuição pode ser confundidade com igualdade quando observamos o código. Por exemplo, não faça uso do codigo a seguir:

if (x = y) {/* statemenet here */}

Se você precisar usar uma atribuição em uma expressão condicional, uma pratica comum é colocar parenteses adicionais em torno da atribuição. Por exemplo:

if ((x = y)) {/* statement here */}

Se você não estiver familiarizado com o operador condicional, você pode ler a documentação oficial sobre o operador condicional primeiro.

Declaração While

Uma declaração while executa suas declarações enquanto uma condição especificada avalia como verdadeira. Uma declaração while se parece com a seguinte:

while (condition) statement

Se a condição se tornar verdadeira, a declaração dentro do laço para a execução e o controle para a proxima execução no laço. Se a Condição retornar false, a execução para e passa o controle para a proxima declaraçao do laço while.

Para executar multiplas declarações, use uma declaração em bloco ({...}) para agrupar estas declarações.

Example:

O seguinte laço while itera equanto o n for menor que três:

var n = 0; var x = 0; while (n < 3){n++; x += n; }

Com cada iteração, o laço incrementa n e adiciona esse valor a x. Portanto, x e n recebem os seguintes valores.

  • Após a primeira passagem: n = 1 e x = 1
  • Após a segunda passagem: n = 2 e x = 3
  • Após a terceira passagen: n = 3 e x = 6

Após completar a terceira passagem, a condição n < 3 deixa de ser verdadeira, então o laço é encerrado.

Declaração de Função

Uma definição de função (declarae uma função ou declaração de funções) consiste da palavra chave function, seguida por:

  • O nome da função.
  • Uma lista de argumentos para a função, entre parenteses e separados por virgulas.
  • As declarações JavaScript que definem a função, entre chaves {...}.

Por exemplo, o código a seguir defien uma função simples chamada square:

function square(number) { return number * number; }

A função square recebe um argumento, chamado number. A função consiste de uma instrução que indica para retornar o argumento da função (isto é, number) multiplicado por si mesmo. A instrução return especifica o valor retornado pela função.

return number * number;

Parametros primitivos (tal como number) são passados para a função pelo valor; o valor é passado para a função, mas se a Funçãoalterar o valor dos parametros, esta mudança não é refletida globalmente ou na chamada da função.
